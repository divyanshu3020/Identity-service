# Contributing to Identity Service

First off, thank you for considering contributing to the Identity Service! Your feedback, code, and ideas are highly appreciated. 

Currently, this is a lightweight Email OTP service, but the vision is to expand its capabilities over time.

## Best Practices
1. **Fork & Branch:** Fork the repository and create a new branch for your feature or bug fix.
2. **Keep it Small:** Try to keep your Pull Requests small and focused on a single issue or feature.
3. **Test:** Ensure you test your changes locally using the plug-and-play `docker-compose` setup.

## AI Usage Policy 🤖
**Using AI to write code for your contributions is 100% accepted and encouraged!** 

However, we ask for one simple rule: 
- **Your Commits & PR Descriptions must be written by YOU.** Please explain *why* and *how* the code works in your own natural human language so we can properly understand and review your thought process.

## How to Submit a Pull Request
1. Commit your changes with a clear, descriptive message.
2. Push to your fork and submit a Pull Request against the `main` branch.
3. Briefly describe the changes you made in the PR description.

Thank you for helping make this service better for everyone!
